# Image Track Decision Tree

4단계 image phase(`graphic-designer`)에서 `info-img` 카드의 이미지를 어떤 방식으로 만들지 결정하는 룰.

현재 운영 트랙은 **웹 무료이미지 / 실사 합성 / 3D 그래픽 / 텍스트형 전환**이다.
CSS 목업은 사용하지 않는다. 일러스트는 보관 트랙이며, 사용자가 명시적으로
요청할 때만 꺼낸다.

---

## 기본 원칙

이미지는 본문을 대신 설명하지 않는다. `info-img`의 상단 이미지는 하단 텍스트가
말하는 내용을 빠르게 감각화하는 보조 장치다.

- 한 카드에는 한 장면, 한 오브젝트, 한 비유만 둔다.
- 이미지 안에는 한글 문장, 로고, 정확한 수치, 카드 제목을 넣지 않는다.
- 정확한 정보는 HTML 텍스트가 담당한다.
- 이미지가 애매하면 `info-text`로 전환한다.

---

## 트랙 정의

| Track | Use When | Owner | Reference |
|---|---|---|---|
| `web-free-image` | 실제 사물·공간·사람 행동이 바로 필요한 경우 | Codex | `info/web-free-image-system.md` |
| `reality-composite` | 사진 같은 질감이 필요하지만 장면을 직접 구성해야 하는 경우 | Codex | `styles/reality-composite.md` |
| `3d-graphic` | 추상 개념을 하나의 상징 오브젝트나 캐릭터 장면으로 만들 때 | Codex | `styles/3d-graphic.md` |
| `photo-composite` | 무료 이미지 기반에 오브젝트/소품을 더해 의미를 강화할 때 | Codex | `styles/reality-composite.md` |
| `text-led` | 이미지가 메시지를 흐리거나 과해지는 경우 | Codex | `card-layout-system` `info-text` |

Parked:

- `illustration` — 지금 기본 트랙에서 제외. 기록은 유지하되 사용자 요청 시에만 사용.
- `css-mockup` — 제거. 데이터·UI 메타포도 이미지가 꼭 필요하지 않으면 `info-text`,
  필요하면 `3d-graphic` 또는 `reality-composite`로 해결한다.

---

## 선택 순서

```text
질문 1: 이미지가 이 카드 이해를 실제로 돕는가?
  NO  → text-led
  YES → 다음

질문 2: 실제 사물/공간/행동 사진만으로 충분한가?
  YES → web-free-image
  NO  → 다음

질문 3: 현실감 있는 장면을 새로 구성해야 하는가?
  YES → reality-composite 또는 photo-composite
  NO  → 다음

질문 4: 하나의 상징 오브젝트/미니 장면으로 더 잘 읽히는가?
  YES → 3d-graphic
  NO  → text-led
```

## 스타일별 판단 기준

### 1. Web Free Image

무료 이미지가 그대로 메시지를 받쳐줄 때 사용한다.

Best:
- 영수증, 계산기, 노트북, 택배 박스, 쇼핑백, 매장, 문서
- 구체적인 업종·상황을 보여주는 장면

Avoid:
- 브랜드 로고가 크게 보이는 사진
- 얼굴이 너무 선명한 인물 사진
- 메시지보다 분위기만 강한 추상 사진

### 2. Reality Composite

실제 사진 같은 질감과 조명이 필요하지만, 장면을 직접 설계해야 할 때 사용한다.

Best:
- 세금, 정산, 수수료, 비용 누수, 정책 변화
- 현실적인 책상·사무실·상점 위에 상징 오브젝트를 배치하는 장면

Avoid:
- 과장된 재난/공포 연출
- 문서 안에 읽히는 글자 생성
- 정확한 숫자를 이미지에 맡기는 구성

### 3. 3D Graphic

시각 비유를 더 부드럽고 브랜드답게 만들 때 사용한다.

Best:
- 마감일, 비교, 선택지, 흐름, 누수, 축적, 자동화
- 하나의 3D 오브젝트나 작은 장면으로 끝나는 카드
- 브랜드/플랫폼 경쟁처럼 캐릭터 표정과 동작이 관계를 설명하는 카드

Avoid:
- 조용한 설명 카드에서 너무 많은 소품
- 경쟁/뉴스형인데도 작은 피규어를 책상 위에 세워두기만 하는 장면
- 브랜드 단서를 흰 스티커/라벨처럼 붙이는 방식

Subtypes:

- `quiet-object`: 작은 오브젝트, 소품 1~3개, 차분한 정보 보조 이미지
- `cinematic-mascot-action`: 큰 캐릭터, 강한 표정, 추격/방어/수리 같은 동작,
  배경 세계관과 모션이 있는 이미지

### 4. Text-Led

이미지가 없어야 정보가 더 잘 읽히는 카드다.

Best:
- 신고 기준, 예외 조건, 체크리스트, 법적 문장
- 본문 디테일이 핵심인 카드

Use:
- `info-text` 레이아웃
- Tossface 이모지는 필요할 때만 한 개
- `h2/body` 토큰 유지

## 시리즈 운영 룰

1. 모든 info 카드가 같은 포맷이면 대표 1장을 먼저 승인받고 나머지에 적용한다.
2. `info-img`와 `info-text`가 섞이면 각각 대표 1장씩 먼저 맞춘다.
3. 실제 이미지를 아직 만들지 않을 때는 `03_info_img.png`의 체커보드
   플레이스홀더를 사용한다.
4. 최종 자산과 출처는 `output/<slug>/assets/`와 `_image-plan.md`에 기록한다.

## 톤 통일 장치

서로 다른 이미지 트랙이 섞여도 한 세트로 보이게 다음을 지킨다.

- 공통 다크 베이스: `#0D0D0D`
- 공통 컬러 액센트: 브랜드 액센트 블루 `#0B78F8`
- 하단 텍스트 박스는 카드 레이아웃 시스템 기준 유지
- 이미지 안의 텍스트·로고·수치 의존 금지
- 사진/합성/조용한 3D는 하단으로 갈수록 조용해지는 구도 선호
- 경쟁/뉴스형 3D는 브랜드 색상과 장면 밀도를 허용하되, 텍스트는 HTML이 담당
