# Research Preferences

Durable preferences for topic research and angle selection.

## Audience

- Default audience: beginner and early-stage ecommerce sellers.
- A topic is useful only if a seller would lose something practical by ignoring
  it.

## Preferred Angles

- Translate news, policy, tools, and market changes into seller decisions.
- Prefer concrete operational usefulness over broad commentary.
- Prefer "what should I check/change/do now?" endings.
- Treat Brand's edge as seller pain digging: margin confusion, sourcing,
  detail-page behavior, lonely operators, consignment vs buying, and
  early-stage execution mistakes.

## Avoid

- Generic trend summaries.
- Topics that only sound interesting but do not change seller behavior.
- Overly promotional Brand mentions before the useful content lands.
- Unsupported claims about market direction, platform policy, prices, or tool
  performance.

## Learning Loop

- If the user accepts a topic, record the topic and why it worked in
  `topic-memory.md`.
- If the user rejects a topic, record the rejection reason in
  `topic-memory.md`.
- If the user repeatedly favors or rejects a source type, update
  `source-preferences.md`.

## 근거 원칙 — "데이터 1 + 사례 2"

한 시리즈에서 인용하는 근거 총합은 **데이터 1개 + 사례 2개 = 최소 3개**.

- **데이터:** 시장 규모, 점유율, 가격·수수료, 거래액 등 수치 1개. 출처와 발행일 필수.
- **사례:** 실제 도구·플랫폼·셀러 사례 2개. 고유명사 + 1줄 설명.
- 추측·일반론 금지. 근거가 부족하면 `WebSearch` / `WebFetch`로 보강 후 작성.

근거가 3개 미만이면 카드 시리즈로 가지 않는다 — 더 좁은 각도를 다시 잡거나, 데이터·사례를 추가 수집한다.

## 자료 신선도 — 30일 이내 우선

- 트렌드·뉴스·정책 변경은 **최근 30일 이내** 자료를 우선.
- 30일이 넘은 자료라도 가치가 있으면 인용 가능, 단 "왜 지금 다시 보는지"의 각도가 명확해야 함.
- 발행일이 불명확한 블로그·커뮤니티 글은 인용 약화. 공식 발표·언론·플랫폼 공지를 1순위.

## 다양성 우선 — 페르소나 3축에 갇히지 않기

`knowledge/persona.md`의 정보 채널 3축(AI 도구 / 유튜버 / 커뮤니티)은 **관찰된 예시이지 강제가 아니다.**

- 같은 축에서 비슷한 주제 후보를 복수 내지 않는다.
- 서로 다른 각도 2~3개를 한 번에 제시 — 정책·도구·운영·세금·물류·해외진출·계절성 등 폭넓게.
- 페르소나가 "반응할 만한" 주제라면 3축 밖이어도 후보로 올린다.
